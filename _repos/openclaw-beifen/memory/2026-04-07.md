- NodeSeek 自动签到体系已基本成型：
  - 使用 Python + cloudscraper 直接调用 `/api/attendance`，不再依赖浏览器点击。
  - Cookie 存在 `scripts/nodeseek_cookies.env`，用户会周期性手动刷新并发给我更新。
  - 每天 00:01 北京时间自动签到，00:03 通过 `telegram_notify.py` 向用户推送极简战报。
  - 推送格式已多轮调整为纯文本 + Emoji，无 Markdown 符号、无括号注释，用户满意当前格式。
  - 推送脚本增加了 Cookie 有效性检测与 Cookie 失效报警机制。

- 与用户约定的体验标准：
  - 战报必须包含：日期、状态、今日奖励鸡腿数、当前总数、下次执行时间、推送时间。
  - 不允许再出现“签到成功但实际没拿到鸡腿”的情况：
    - 只有在 API 返回 `success=true` 且包含 `gain` 字段时才标记为成功；
    - 否则标记为“可疑”或“失败”，并在战报中说明原因。
  - 不在非约定时间（尤其是白天）主动推送测试消息，以免打扰用户。

- OpenClaw Zero Token 项目：
  - 已在 VPS 上克隆并完成依赖安装与构建，环境就绪但尚未配置具体网站的 Cookie/凭证。
  - 用户有兴趣先尝试豆包 Doubao，通过手动提供 Cookie 注入方式来启用免费模型网关，计划待后续继续。

- 用户风格与偏好：
  - 对信息格式和整洁度要求高，讨厌“乱糟糟”的输出和冗余说明，更偏好极简、状态卡式的信息展示。
  - 可以接受为了稳定而使用 Cookie 方案，但也愿意在不影响现有稳定性的前提下尝试高难度实验（如账密+2Captcha 登录）。
  - 希望系统能在出现异常（例如 Cookie 失效、签到失败）时第一时间主动告警，而不是事后解释。
